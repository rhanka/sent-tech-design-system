# Audit accessibilité — composants ajoutés cette session

Source de référence : `packages/components-svelte/src/lib/*.svelte` (Svelte = implémentation pilote).
Périmètre : layout & groupes/identité (Avatar, AvatarGroup, ButtonGroup, CheckboxGroup, RadioGroup, Typography, Collapsible, Stepper), spécialisés (Rating, TimePicker, Calendar, SlideIndicator, Autosave), primitives (Portal, Popper), UAT8 (SelectableRow, SelectableList).

Audit **read-only**. Aucune modification de code. Seul ce fichier est créé.

Sévérité : **Bloquant** (empêche l'usage clavier/SR) · **Majeur** (dégrade fortement) · **Mineur** (amélioration).

---

## Problèmes relevés

| Composant | Problème a11y | Sévérité | Critère WCAG | Correctif |
|---|---|---|---|---|
| **TimePicker** | Pattern `combobox`/`listbox` cassé au clavier : l'input a `role="combobox" aria-controls/aria-expanded` mais à l'ouverture le focus ne descend PAS dans la liste, aucune flèche Haut/Bas/Home/End ne déplace l'option active (`onPanelKeyDown` ne gère que Escape). Les options ne sont atteignables qu'en Tab à travers tous les créneaux (jusqu'à 96 boutons). | Bloquant | 2.1.1 Clavier · 4.1.2 Nom/rôle/valeur · 1.3.1 | Implémenter la nav clavier listbox : ArrowDown/Up déplacent un `aria-activedescendant` (ou roving focus) sur les `option`, Home/End aux extrémités, Enter sélectionne. À l'ouverture, focus sur l'option sélectionnée. Ne pas laisser chaque option dans l'ordre de tabulation. |
| **TimePicker** | `<input readonly role="combobox">` : un combobox ne s'ouvre pas au clavier. `onclick=toggleOpen` ouvre à la souris, mais Entrée/Espace/Flèche-bas sur l'input ne déclenchent rien (pas de `onkeydown`). | Bloquant | 2.1.1 Clavier | Ajouter `onkeydown` sur l'input : ArrowDown/Enter/Espace ouvrent et placent le focus dans la liste. |
| **Calendar** | `role="grid"` mais navigation clavier de grille absente : seuls PageUp/PageDown (changement de mois) sont gérés sur le conteneur. Aucune flèche ne déplace le focus de jour en jour ; les 42 boutons-jours sont tous dans l'ordre de tabulation (pas de roving tabindex, pas d'`aria-activedescendant`). Le `tabindex="-1"` sur le grid ne suffit pas. | Bloquant | 2.1.1 Clavier · 4.1.2 | Roving tabindex sur les cellules (un seul jour `tabindex=0`, les autres `-1`) + ArrowLeft/Right/Up/Down (±1/±7 jours), Home/End (début/fin de semaine), gestion du passage de mois quand on dépasse le bord. |
| **Calendar** | Cellules `role="gridcell"` placées directement dans un conteneur `.st-calendar__days` sans `role="row"`. Une `grid` exige la hiérarchie grid → row → gridcell ; ici les gridcell sont orphelines de `row`. La ligne d'en-têtes a bien `role="row"`, pas les semaines. | Majeur | 1.3.1 Info & relations · 4.1.2 | Envelopper chaque semaine de 7 jours dans un `role="row"`, ou retirer le rôle grid au profit d'un pattern plus simple si la nav grille n'est pas implémentée. |
| **SlideIndicator** | `role="tablist"` + `role="tab"` sans `tabpanel` associé ni `aria-controls`. Un `tab` orphelin (sans panneau contrôlé) trompe le lecteur d'écran qui annonce « onglet » pour un simple indicateur de pagination. | Majeur | 4.1.2 Nom/rôle/valeur · 1.3.1 | Soit fournir `aria-controls` pointant le panneau de slide et exposer le panneau en `tabpanel`, soit abandonner tablist/tab au profit d'un `role="group"` de boutons (ou pattern carrousel ARIA) — l'indicateur de carrousel n'est pas un tablist. |
| **SlideIndicator** | `aria-current="true"` posé en plus d'`aria-selected` sur un `role="tab"`. Sur un tab, l'état courant s'exprime par `aria-selected` ; `aria-current` y est redondant/non standard. | Mineur | 4.1.2 | Retirer `aria-current` sur le tab (conserver `aria-selected`). Si on garde un pattern bouton/group, `aria-current="true"` devient au contraire pertinent. |
| **Rating** | `aria-checked` incohérent avec les demi-étoiles : `aria-checked` se base sur `Math.ceil(value) === star`, donc à value=2.5 (allowHalf) l'étoile 3 est annoncée « cochée » alors qu'elle est à moitié. Aucune exposition de la valeur réelle (pas d'`aria-valuenow`/texte). | Majeur | 4.1.2 Nom/rôle/valeur | Soit exposer un `slider` (`aria-valuemin/max/now/valuetext` « 2,5 / 5 ») mieux adapté aux demi-pas, soit garder radiogroup mais sans demi-étoiles, ou ajouter un libellé live annonçant la valeur fractionnaire. |
| **Rating** | Roving tabindex : à value=0 c'est la 1re étoile qui est `tabindex=0`, mais aucune n'est `aria-checked="true"` — cohérent. En revanche `aria-checked="false"` est posé sur toutes les étoiles non sélectionnées : pour un radiogroup à note partielle (ex. 3/5), les étoiles 1-2 « pleines » sont annoncées non cochées, ce qui contredit le visuel rempli. | Mineur | 1.3.1 · 4.1.2 | Modèle radio = une seule sélection : ne marquer cochée que l'étoile == valeur (déjà le cas) ; ajouter un nom accessible décrivant le remplissage cumulé (« 3 étoiles sur 5 ») plutôt que des labels « 1 / 5 » par bouton qui ne disent pas l'état global. |
| **Rating** | En `readonly`, les étoiles sont rendues `disabled` : la note n'est plus du tout focusable ni lisible au clavier/Tab, alors qu'un affichage en lecture seule doit rester perceptible. `aria-readonly` est posé mais le `disabled` retire les boutons de l'arbre d'accessibilité interactif. | Mineur | 1.3.1 | En readonly, préférer un rendu non interactif avec texte accessible (« Note : 4 sur 5 ») plutôt que des boutons `disabled` ; ou `img`/`aria-label` global. |
| **Collapsible** | Le `role="region"` n'est généré dans le DOM que lorsque `open` est vrai (`{#if open}`). `aria-controls` du bouton pointe alors un id absent quand fermé. Acceptable, mais certains SR n'annoncent pas correctement le rapport. | Mineur | 1.3.1 · 4.1.2 | Conserver le conteneur région dans le DOM (masqué via `hidden`/CSS) pour que `aria-controls` cible un élément toujours présent, ou accepter le pattern (largement toléré). |
| **Popper** | Aucun focus management ni `focus-trap` : le panneau flottant (utilisé pour menus/popovers/dialogues) ne capture pas le focus, ne le restitue pas à l'ancre à la fermeture, et n'a ni rôle ni `aria-*` (c'est un `div` nu). Si un overlay modal est bâti dessus, le focus reste sous le panneau. | Majeur | 2.4.3 Ordre de focus · 2.1.2 Pas de piège clavier (inverse : fuite) · 4.1.2 | Documenter explicitement que Popper est un primitif de positionnement « sans a11y » et que le consommateur (Menu, Tooltip, Dialog) doit ajouter rôle, gestion focus/trap et Escape. Idéalement exposer des hooks optionnels (trapFocus, returnFocus, role). |
| **Popper** | Pas de fermeture clavier intégrée : aucun `Escape` ni gestion du focus sortant au niveau du primitif. La fermeture au clic extérieur n'est pas non plus gérée ici. | Mineur | 2.1.1 | Idem ci-dessus : à la charge du consommateur ; le préciser dans la doc/API du primitif. |
| **Stepper** | En mode `clickable`, le bouton d'étape n'expose pas son état désactivé/non atteignable : on peut cliquer une étape « upcoming ». `aria-current="step"` est sur le `<li>`, pas sur le bouton ; le bouton n'a ni `aria-current` ni `disabled` selon l'état. | Mineur | 4.1.2 | Sur le bouton d'étape, refléter l'état : `aria-current="step"` quand current, et désactiver (ou `aria-disabled`) les étapes non navigables si la logique l'exige. |
| **CheckboxGroup** | `helperText` du groupe (description sous la légende) n'est pas relié programmatiquement aux cases via `aria-describedby` : la description n'est pas annoncée. `disabled` du fieldset cascade bien nativement. | Mineur | 1.3.1 · 3.3.2 | Donner un id au `<p>` d'aide et le référencer en `aria-describedby` sur le fieldset (ou sur chaque input). |
| **RadioGroup** | Idem CheckboxGroup : `helperText` non relié en `aria-describedby`. Par ailleurs le RadioGroup repose entièrement sur les radios natifs (bon point : nav flèches native), mais la légende d'aide reste non associée. | Mineur | 1.3.1 · 3.3.2 | Relier le helperText via `aria-describedby`. |
| **ButtonGroup** | `role="group"` sans `aria-label` quand la prop `label` est absente : un groupe sans nom accessible n'apporte rien au SR (groupe anonyme). | Mineur | 1.3.1 · 4.1.2 | Rendre `label` requis quand `role="group"` est posé, ou n'ajouter `role="group"` que si un nom est fourni. |
| **AvatarGroup** | Le jeton de débordement « +N » porte `aria-label="+N"` mais sa signification (N personnes de plus) n'est pas explicite ; le groupe lui-même n'a pas de nom ni de rôle de liste. Les Avatar enfants sont des `img` avec label, OK. | Mineur | 1.3.1 | Donner au « +N » un libellé parlant (« 3 de plus ») et envisager `role="group"`/`list` avec un nom sur le conteneur. |
| **Calendar** | `aria-current="date"` : valeur valide. `aria-live="polite"` sur le label de mois statique se déclenche à chaque changement — correct, mais le changement de mois ne déplace pas le focus, donc l'utilisateur clavier de la grille (une fois la nav implémentée) perd le repère. | Mineur | 2.4.3 | Lors d'un changement de mois au clavier, recaler le focus sur un jour du nouveau mois. |
| **SelectableRow / SelectableList** | Bien conçu (listbox/option, roving tabindex via contexte, Home/End/flèches, `aria-multiselectable`). Limite : la nav flèches **clampe sans wrap** et ne saute pas les rows `disabled` — une row désactivée intercalée n'est pas focusable (tabindex -1) mais `navigate` peut cibler son index et appeler `.focus()` sur un élément non focusable, bloquant l'avance. | Majeur | 2.1.1 Clavier | Dans `navigate`, sauter les entrées dont la row est disabled (chercher le prochain index focusable) ; sinon le focus peut « caler » sur une row désactivée. |
| **SelectableRow** | Standalone : `role="option"` par défaut hors de tout `listbox` parent → `option` orpheline (un `option` doit vivre dans un `listbox`). Le commentaire le justifie pour l'usage en liste, mais une row isolée expose un rôle invalide en contexte. | Mineur | 1.3.1 · 4.1.2 | Pour la row standalone, défaut `role` plus neutre (`button`/`checkbox` selon l'usage) plutôt qu'`option`, ou exiger un wrapper listbox. |

## Bons points (pour mémoire, non comptés)

- **Avatar** : `role="img"` + `aria-label`, initiales `aria-hidden`. Correct.
- **CheckboxGroup / RadioGroup** : `fieldset`/`legend` natifs, cases natives (focus + nav flèches radio gratuites), `disabled` cascade. Très bon socle.
- **Autosave** : `role` dynamique `status`/`alert` selon l'état, `aria-live="polite"`, `prefers-reduced-motion` respecté sur le spinner. Solide.
- **Collapsible** : `aria-expanded` + `aria-controls` + `aria-labelledby`, bouton natif, focus-visible. Bon.
- **Typography** : sémantique d'en-têtes via `as`/variante ; attention seulement à ce que le consommateur ne casse pas l'ordre des niveaux (hors périmètre composant).
- **SelectableList/Row** : meilleur élève côté clavier (roving, contexte, ARIA listbox complet) — seul le saut des disabled manque.

## Top-10 priorisé

1. **TimePicker — nav clavier listbox absente** (Bloquant) : combobox inutilisable au clavier. *(2.1.1 / 4.1.2)*
2. **Calendar — nav clavier de grille absente** (Bloquant) : grille de dates non parcourable aux flèches. *(2.1.1 / 4.1.2)*
3. **TimePicker — input combobox non ouvrable au clavier** (Bloquant) : pas de `onkeydown`. *(2.1.1)*
4. **SelectableList — `navigate` ne saute pas les rows disabled** (Majeur) : focus qui cale, piège potentiel. *(2.1.1)*
5. **Popper — aucun focus management / trap / Escape** (Majeur) : fuite de focus si overlay modal. *(2.4.3 / 4.1.2)*
6. **Calendar — `gridcell` sans `role="row"` parent** (Majeur) : hiérarchie grid invalide. *(1.3.1 / 4.1.2)*
7. **SlideIndicator — `tab`/`tablist` sans `tabpanel`/`aria-controls`** (Majeur) : rôle d'onglet trompeur. *(4.1.2)*
8. **Rating — `aria-checked` faux avec demi-étoiles** (Majeur) : valeur annoncée incorrecte. *(4.1.2)*
9. **Stepper (clickable) — état du bouton d'étape non exposé** (Mineur) : `aria-current`/disabled manquants sur le bouton. *(4.1.2)*
10. **Checkbox/RadioGroup — `helperText` non relié en `aria-describedby`** (Mineur) : description non annoncée. *(1.3.1 / 3.3.2)*
